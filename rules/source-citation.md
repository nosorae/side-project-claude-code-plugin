# 소스 인용 규칙

외부 정보를 기반으로 문서를 작성할 때 출처를 명시한다.

## 소스 등급

| 등급 | 설명 | 예시 |
|------|------|------|
| **Tier 1** (신뢰) | 공식 문서, 공식 블로그, 학술 자료, 정부 통계 | Android Developers, MDN, 논문, 통계청 |
| **Tier 2** (참고) | 커뮤니티 검증 자료, 유명 기술 블로그, 리뷰 플랫폼 | Stack Overflow, Product Hunt, G2, Reddit (고추천) |
| **Tier 3** (보조) | 개인 블로그, 비검증 포럼 글, SNS | 개인 미디엄, 트위터, 비추천 Reddit 글 |

## 인용 원칙

- Tier 1 소스만 사용한 경우: 그대로 인용
- Tier 2 소스를 사용한 경우: 가능하면 Tier 1로 교차검증
- Tier 3 소스만 있는 경우: **반드시** Tier 1 또는 Tier 2로 교차검증. 불가하면 `(미검증)` 표시
- 출처가 없는 주장은 `(추정)` 또는 `(경험 기반)` 표시

## 인용 형식

문서 내 인용:
```markdown
시장 규모는 약 5,000억원이다. ([출처: 통계청 2025](URL))
```

문서 하단 참고자료:
```markdown
## 참고자료
- [제목](URL) — Tier 1, 요약
- [제목](URL) — Tier 2, 요약
```

## 적용 범위

- 모든 스킬에서 외부 조사 결과를 문서에 기록할 때 적용
- 코드 주석이나 커밋 메시지에는 적용하지 않음
