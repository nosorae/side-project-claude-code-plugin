# 품질 체크리스트 & 트러블슈팅

## Step 1 (3인 토론)
- [ ] 3개 에이전트가 독립적으로 조사했는가?
- [ ] 시장 옹호자가 시장 규모 데이터를 근거로 제시했는가?
- [ ] 경쟁 분석가가 경쟁 앱 3개 이상 조사했는가?
- [ ] 악마의 변호인이 실패 리스크를 구체적으로 지적했는가?
- [ ] opus가 3개 관점을 종합하여 편향 없이 판정했는가?
- [ ] 참고 자료 링크가 포함되었는가?

## Step 2 (3인 토론)
- [ ] 사용자 옹호자가 페인 포인트 기반으로 기능을 제안했는가?
- [ ] 비즈니스 전략가가 수익 모델을 근거와 함께 추천했는가?
- [ ] 기술 현실주의자가 구현 가능성을 검증했는가?
- [ ] 쟁점 테이블에서 3개 관점 충돌이 드러나는가?

## Step 3
- [ ] 화면이 5개 이하로 제한되었는가?
- [ ] 기술 스택이 AI First 원칙에 맞는가?
- [ ] Must Have / Nice to Have가 명확히 분리되었는가?

## Step 4
- [ ] 핵심 유저 플로우가 최단 경로인가?
- [ ] 3초 룰을 통과하는가?

## Step 5
- [ ] 가독성 규칙 준수 + Git 커밋 메시지 명확

## 트러블슈팅

| 상황 | 해결 |
|------|------|
| 경쟁 앱 정보 부족 | 영어 키워드로 재검색, "Google Play Store [keyword] top apps" |
| 시장 규모 추정 어려움 | 유사 카테고리 참고, TAM × ARPU 계산, "추정치" 명시 |
| MVP 범위 너무 큼 | "없으면 안 되는 것"만 남기기, v2로 과감히 미루기 |
| 유저 플로우 복잡 | Happy Path만 먼저 정의, 부가 기능은 별도 플로우 |
