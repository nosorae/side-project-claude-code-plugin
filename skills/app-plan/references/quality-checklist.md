# 품질 체크리스트 & 트러블슈팅

## 품질 체크리스트

각 단계 완료 후 확인:

### Step 1 (3인 토론)
- [ ] 3개 에이전트가 **독립적으로** 조사했는가? (서로 참조 금지)
- [ ] 시장 옹호자가 시장 규모 데이터를 근거로 제시했는가?
- [ ] 경쟁 분석가가 경쟁 앱 3개 이상 조사했는가?
- [ ] 악마의 변호인이 실패 리스크를 구체적으로 지적했는가?
- [ ] opus가 3개 관점을 종합하여 편향 없이 판정했는가?
- [ ] 참고 자료 링크가 포함되었는가?

### Step 2 (3인 토론)
- [ ] 사용자 옹호자가 페인 포인트 기반으로 기능을 제안했는가?
- [ ] 비즈니스 전략가가 수익 모델을 근거와 함께 추천했는가?
- [ ] 기술 현실주의자가 기술적 구현 가능성을 검증했는가?
- [ ] 쟁점 테이블에서 3개 관점 충돌이 명시적으로 드러나는가?
- [ ] opus가 최적 교집합을 도출했는가?

### Step 3
- [ ] 화면이 5개 이하로 제한되었는가?
- [ ] 기술 스택이 AI First 원칙에 맞는가?
- [ ] 우선순위 분류가 명확한가?
- [ ] Must Have와 Nice to Have가 명확히 분리되었는가?

### Step 4
- [ ] 핵심 유저 플로우가 최단 경로인가?
- [ ] 화면별 액션이 구체적인가?
- [ ] 3초 룰을 통과하는가?
- [ ] 예외 처리가 포함되었는가?

### Step 5
- [ ] 가독성 규칙을 준수했는가?
- [ ] Git 커밋 메시지가 명확한가?

## 트러블슈팅

### 경쟁 앱 정보가 부족할 때
1. 검색어를 더 구체적으로 조정
2. 영어 키워드로 재검색
3. "Google Play Store [keyword] top apps 2026" 검색
4. 직접 구글플레이 링크 확인 요청

### 시장 규모 추정이 어려울 때
1. 유사 카테고리 시장 규모 참고
2. 타겟 사용자 수 × 예상 ARPU로 계산
3. "추정치" 명시 + 근거 설명
4. 보수적으로 계산

### MVP 범위가 너무 클 때
1. "없으면 안 되는 것"만 남기기
2. 사용자와 범위 재조정 논의
3. v2 Features로 과감히 미루기
4. "핵심 가치만 전달하는 최소 버전" 강조

### 유저 플로우가 너무 복잡할 때
1. "핵심 가치 체험"까지만 플로우 정의
2. 부가 기능은 별도 플로우로 분리
3. 최단 경로(Happy Path)만 먼저 정의
4. 예외 처리는 나중에
