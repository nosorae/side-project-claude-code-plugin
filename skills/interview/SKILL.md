# Interview (요구사항 인터뷰)

사용자의 앱 아이디어를 점진적 인터뷰로 구체화하여, `/app-plan` 스킬이 바로 소비할 수 있는 구조화된 요구사항 문서를 생성합니다.

## 트리거 조건

다음과 같은 요청이 들어올 때 자동 발동:
- "/interview", "인터뷰", "요구사항 정리"
- "앱 아이디어 정리해줘", "기획 전에 정리 좀"
- "뭘 만들고 싶은지 물어봐줘"

## 핵심 원칙

- **최대 6라운드**로 제한한다. 사용자를 지치게 하지 않는다.
- **한 번에 1~2개 질문**만 한다. 질문 폭탄을 던지지 않는다.
- **추론 우선, 확인 후순위**: 이전 답변에서 유추할 수 있는 것은 먼저 추론하고 "~이렇게 이해했는데 맞나요?" 형태로 확인한다.
- **각 라운드에 주제 초점**이 있다. 산만하게 여러 주제를 섞지 않는다.
- 사용자가 충분한 정보를 이미 제공했으면 남은 라운드를 스킵하고 바로 요약으로 넘어간다.

---

## 실행 단계

### Round 1: 아이디어 파악 (오픈 질문)

**목적:** 사용자의 머릿속 아이디어를 자유롭게 꺼내기

**질문 예시:**
- "어떤 앱을 만들고 싶으신가요? 떠오르는 대로 편하게 말씀해주세요."

**AI 행동:**
- 사용자의 답변에서 핵심 키워드와 도메인을 추출한다
- 다음 라운드 질문 방향을 결정한다
- 이미 언급된 정보(목적, 타겟 등)는 체크하여 중복 질문하지 않는다

---

### Round 2: 목적과 동기

**목적:** 왜 이 앱을 만들려는지, 어떤 문제를 해결하는지 이해

**질문 예시 (Round 1 답변에 따라 선택):**
- "이 앱이 해결하려는 가장 큰 문제(불편함)는 뭔가요?"
- "이 앱이 있으면 사용자의 하루가 어떻게 달라지나요?"

**AI 행동:**
- Round 1에서 동기가 이미 드러났으면 "~라는 문제를 해결하려는 것으로 이해했는데 맞나요?" 형태로 확인만 한다
- 확인되면 이 라운드를 빠르게 넘긴다

---

### Round 3: 타겟 사용자

**목적:** 누가 쓸 앱인지 구체화

**질문 예시:**
- "이 앱을 가장 먼저 쓸 사람은 누구인가요? (예: 20대 직장인, 대학생, 주부 등)"
- "그 사람들이 지금은 이 문제를 어떻게 해결하고 있나요?"

**AI 행동:**
- 앞선 답변에서 타겟이 암시된 경우 "타겟은 ~인 것 같은데, 맞나요?" 형태로 확인
- 현재 대안(경쟁 상황)에 대한 힌트를 수집한다

---

### Round 4: 핵심 기능과 참고 앱

**목적:** 앱의 핵심 기능 1~3개와 벤치마크 파악

**질문 예시:**
- "이 앱에서 사용자가 가장 자주 할 행동(핵심 기능) 1~2개는 뭔가요?"
- "참고하거나 좋아하는 앱이 있나요? 어떤 점이 좋았나요?"

**AI 행동:**
- 기능이 너무 많으면 "이 중에서 하나만 남긴다면?" 으로 좁힌다
- 참고 앱에서 차별점 힌트를 추출한다

---

### Round 5: 기술 제약과 수익 모델

**목적:** 현실적 제약 조건 파악

**질문 예시 (필요한 것만 선택):**
- "플랫폼은 어디를 생각하세요? (안드로이드, iOS, 웹, 상관없음)"
- "수익 모델에 대해 생각해본 게 있나요? (광고, 구독, 일회성 결제, 무료 등)"
- "기술적으로 꼭 필요한 기능이 있나요? (카메라, GPS, 알림 등)"

**AI 행동:**
- 수익 모델을 아직 안 정했으면 "일단 넘어가도 됩니다, app-plan에서 분석해드릴게요"로 부담을 덜어준다
- 기술 제약이 없으면 이 라운드를 빠르게 넘긴다

---

### Round 6: 보충 및 우선순위

**목적:** 빠진 부분 채우기, 우선순위 확인

**질문 예시:**
- "혹시 꼭 포함해야 하는 기능이나, 반대로 절대 빼야 하는 기능이 있나요?"
- "MVP에서 가장 중요한 건 뭔가요? (속도, 완성도, 특정 기능 등)"

**AI 행동:**
- 앞선 라운드에서 충분히 수집했으면 이 라운드를 스킵한다
- "충분한 정보가 모인 것 같습니다. 정리해볼게요." 로 전환

---

## 요약 및 확인

모든 라운드(또는 충분한 정보 수집) 후 구조화된 요약을 생성하여 사용자에게 제시한다.

**요약 형식:**

```markdown
## 인터뷰 요약

### 앱 아이디어
{한 문장 요약}

### 해결하려는 문제
{핵심 문제/페인 포인트}

### 타겟 사용자
- **주요 타겟:** {구체적 페르소나}
- **현재 대안:** {지금 사용자가 쓰는 방법}

### 핵심 기능 (우선순위 순)
1. {기능 1} — {이유}
2. {기능 2} — {이유}
3. {기능 3} — {이유}

### 참고 앱
- {앱 이름}: {참고할 점} / {차별화할 점}

### 기술 제약
- **플랫폼:** {안드로이드/iOS/웹/크로스플랫폼}
- **필수 네이티브 기능:** {카메라, GPS 등 또는 없음}
- **기타 제약:** {있으면 기재}

### 수익 모델
- **희망 모델:** {구독/광고/무료 등 또는 미정}
- **참고:** {사용자가 언급한 가격대 등}

### 추가 요구사항
- {MVP 우선순위, 제외 기능, 톤앤매너 등}
```

**사용자에게 확인 요청:**

```
위와 같이 정리했습니다. 수정하거나 추가할 내용이 있으면 말씀해주세요.

확인되면 이 내용을 기반으로:
- `/app-plan`으로 기획을 시작하거나
- 추가 보완이 필요한 부분만 더 논의할 수 있습니다.
```

## 문서 저장

사용자가 요약을 확인하면:

1. `docs/ssot/prd/interview-notes.md` 파일로 저장한다
2. 파일 상단에 메타 정보를 포함한다:

```markdown
# 인터뷰 노트

> **작성일:** YYYY-MM-DD
> **상태:** 확인 완료
> **다음 단계:** /app-plan

---

{위 요약 내용 전체}
```

3. Git commit + push 한다

## 조기 종료 조건

다음 경우 남은 라운드를 스킵하고 바로 요약으로 넘어간다:
- 사용자가 Round 1에서 이미 대부분의 정보를 상세하게 제공한 경우
- 사용자가 "이 정도면 됐어", "정리해줘" 등으로 종료를 요청한 경우
- 4라운드 이상 진행 후 추가 질문할 내용이 없는 경우

---

## /app-plan 연계

인터뷰 결과는 `/app-plan`의 입력으로 직접 사용된다:

| 인터뷰 항목 | app-plan 활용 단계 |
|-------------|-------------------|
| 앱 아이디어 + 해결하려는 문제 | Step 1: 아이디어 검증 — 검색 키워드 도출 |
| 타겟 사용자 + 현재 대안 | Step 1: 시장 조사 + 경쟁 분석 |
| 핵심 기능 | Step 2: 핵심 가치 정의 입력 |
| 참고 앱 | Step 1: 경쟁 분석 벤치마크 |
| 기술 제약 + 플랫폼 | Step 3: 기술 스택 결정 |
| 수익 모델 | Step 2: 비즈니스 전략 입력 |
| 추가 요구사항 | Step 3: 우선순위 분류 |

`/app-plan` 실행 시 `docs/ssot/prd/interview-notes.md`가 존재하면 자동으로 읽어 입력으로 사용한다.

## 예시

### Example 1: 정보가 적은 경우

**User says:**
```
앱 하나 만들고 싶은데 인터뷰해줘
```

**AI (Round 1):**
```
좋습니다! 어떤 앱을 만들고 싶으신가요? 떠오르는 대로 편하게 말씀해주세요.
아직 구체적이지 않아도 괜찮습니다.
```

**User:**
```
식단 관리 앱이요. 매일 뭐 먹었는지 기록하고 싶어요.
```

**AI (Round 2):**
```
식단 기록 앱이군요. 두 가지만 여쭤볼게요:

1. 식단 기록의 주된 목적이 뭔가요? (다이어트, 건강 관리, 알러지 추적 등)
2. 기존에 써본 식단 앱이 있다면, 어떤 점이 불편했나요?
```

→ 이런 식으로 6라운드 이내에 요약까지 도달

### Example 2: 정보가 풍부한 경우

**User says:**
```
인터뷰해줘. 20대 직장인 대상 퇴근 후 운동 루틴 공유 앱 만들려고 해.
핵심은 간단한 루틴 체크리스트고, 참고 앱은 Nike Training Club이야.
프리미엄 구독 모델 생각 중이고, 안드로이드 우선이야.
```

**AI (바로 Round 5~6 수준으로 점프):**
```
상당히 구체적으로 정리해주셨네요! 제가 이해한 내용을 확인하겠습니다:

- **목적:** 퇴근 후 운동 루틴을 쉽게 관리하고 공유
- **타겟:** 20대 직장인
- **핵심 기능:** 루틴 체크리스트 (간단함이 핵심)
- **참고 앱:** Nike Training Club
- **수익:** 프리미엄 구독
- **플랫폼:** 안드로이드 우선

한 가지만 더 확인할게요:
"공유" 기능이 MVP에 포함되어야 하나요, 아니면 체크리스트만으로 v1 출시하고 공유는 v2로 미뤄도 될까요?
```

→ 1~2라운드 만에 요약으로 전환

## 품질 체크리스트

인터뷰 완료 시 확인:
- [ ] 6라운드를 초과하지 않았는가?
- [ ] 각 라운드에서 질문이 2개 이하인가?
- [ ] 이전 답변에서 유추 가능한 것을 다시 묻지 않았는가?
- [ ] 요약이 `/app-plan` 입력으로 충분한 정보를 담고 있는가?
- [ ] 사용자가 요약을 확인했는가?
- [ ] `docs/ssot/prd/interview-notes.md`에 저장했는가?

## 관련 스킬

- `ideation` — 아이디어 자체가 없을 때 발굴 (선행)
