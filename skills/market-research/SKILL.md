# Market Research (아이디어 검증)

앱 아이디어를 **3명의 독립 에이전트**가 각자 조사한 후 토론하여 검증하는 스킬입니다.

> **왜 토론인가?** 혼자 조사하면 확증 편향에 빠진다. 서로 다른 관점의 에이전트가 독립적으로 조사하고 반박하면 더 정확한 판단이 나온다.

## 트리거 조건

다음과 같은 요청이 들어올 때 자동 발동:
- "시장 조사해줘", "아이디어 검증해줘"
- "/market-research"
- "이 아이디어 괜찮을까?", "경쟁 분석해줘"

## 실행 단계

### Step 1: 아이디어 수집

**Actions:**

- 사용자에게 앱 아이디어를 물어본다
- 핵심 키워드 추출 (예: "운동 루틴 공유 앱")

### Step 2: 3인 에이전트 병렬 조사 (Task 도구로 동시 실행)

각 에이전트는 WebSearch를 사용하여 **독립적으로 조사**합니다:

**에이전트 A: 시장 옹호자 (Market Advocate)**
- 역할: "이 아이디어가 성공할 수 있는 이유"를 찾는다
- 조사 항목:
  - 시장 규모 (TAM/SAM/SOM)
  - 성장 트렌드 데이터
  - 유사 분야 성공 사례
  - 타겟 사용자 규모와 지불 의향
- 검색 전략: "keyword market size 2026", "keyword app success story revenue"
- **결론 형식**: "이 아이디어는 [근거]로 시장성이 있다/없다"

**에이전트 B: 경쟁 분석가 (Competition Analyst)**
- 역할: 경쟁 현황을 냉정하게 파악한다
- 조사 항목:
  - 경쟁 앱/서비스 5개 이상 (다운로드 수, 평점, 핵심 기능)
  - 사용자 리뷰에서 추출한 불만/개선점
  - 경쟁사가 놓치고 있는 빈틈 (Gap)
  - 시장 포지셔닝 맵 (가격 x 기능 매트릭스)
- **리서치 플랫폼 (프로젝트 타입별 자동 선택)**:
  - **Tier 1 (필수)**: Play Store/App Store 리뷰, Product Hunt, G2/Capterra
  - **Tier 2 (권장)**: Indie Hackers, Reddit (r/SideProject, r/AndroidDev 등), AlternativeTo
  - **Tier 3 (선택)**: Hacker News (Show HN), Crunchbase (무료 티어)
  - 모바일 앱 → Tier 1에서 Play Store/App Store 우선
  - 웹/SaaS → Tier 1에서 Product Hunt, G2 우선
  - `/interview` 결과에 플랫폼 정보가 있으면 해당 플랫폼 우선 조사
- 검색 전략: "keyword app Play Store reviews", "keyword site:producthunt.com", "keyword site:g2.com", "keyword app reviews complaints"
- **소스 등급 적용**: `rules/source-citation.md` 기준으로 Tier 3 소스는 교차검증 필수
- **결론 형식**: "경쟁 상황은 [수준]이며, 진입 기회는 [근거]로 있다/없다"

**에이전트 C: 악마의 변호인 (Devil's Advocate)**
- 역할: "이 아이디어가 실패할 이유"를 적극적으로 찾는다
- 조사 항목:
  - 이 분야에서 실패한 앱/서비스 사례
  - 진입 장벽과 리스크
  - "왜 이미 누군가 안 만들었는가?" 의문
  - 규제/법적 리스크
  - AI 코딩 기반으로 실행 가능한가? 기술적 난이도는?
- 검색 전략: "keyword app failed why", "keyword startup challenge risk"
- **결론 형식**: "이 아이디어의 가장 큰 리스크는 [1, 2, 3]이다"

### Step 3: 토론 및 종합 (opus)

3개 에이전트의 결과를 모아 **opus 모델이 최종 판정**:

```markdown
## 토론 결과

### 시장 옹호자의 주장
[에이전트 A 결론 요약]

### 경쟁 분석가의 주장
[에이전트 B 결론 요약]

### 악마의 변호인의 반박
[에이전트 C 결론 요약]

### 최종 판정

**진행 가치:** ⭐⭐⭐⭐☆ (5점 만점)

| 항목 | 판정 | 근거 |
|------|------|------|
| 시장 규모 | ✅/⚠️/❌ | ... |
| 경쟁 강도 | ✅/⚠️/❌ | ... |
| 차별화 가능성 | ✅/⚠️/❌ | ... |
| 실행 가능성 (AI 코딩 기반) | ✅/⚠️/❌ | ... |
| 수익화 가능성 | ✅/⚠️/❌ | ... |

**핵심 리스크:** ...
**핵심 기회:** ...
**최종 권고:** 진행 / 피봇 제안 / 중단
```

### Step 4: 결과 저장

**Actions:**

1. 결과를 `docs/ssot/prd/YYYY-MM-DD-[앱이름]-시장조사.md`에 저장
2. Git 커밋 + 푸시
3. 사용자에게 결과 요약 + 다음 단계 안내:
   - 진행 권고 → `/app-plan`으로 기획 시작
   - 피봇 제안 → 피봇 방향 제시 후 재검증
   - 중단 권고 → 이유 설명

## 모델 선택 가이드

- 에이전트 A, B, C: `sonnet` (독립 웹 검색 + 분석, 병렬 실행)
- 토론 종합 + 최종 판정: `opus` (다각도 판단, 편향 보정)

## 관련 스킬

- `app-plan` — 검증 통과 후 기획 진행 (후행)
- `interview` — 사용자 인터뷰로 아이디어 구체화 (선택적 선행)
