# Ideation (앱 아이디어 발굴) v2

검증된 인디 해커 방법론 8가지를 **병렬 에이전트**로 동시 실행하고, **수익 검증**과 **안티패턴 필터링**까지 수행하는 데이터 기반 아이디어 발굴 워크플로우.

## 핵심 철학

> **"좋은 아이디어는 발명하는 게 아니라 발견하는 것이다."** — Paul Graham

**기존 ideation 대비 강화점:**
- 8개 에이전트 (수익 검증 + 안티패턴 탐지 추가)
- 이전 아이데이션 결과 자동 중복 제거
- 에이전트당 5개 아이디어 (교차 검증 풀 확대: 18개 → 40개)
- 경쟁 해자(Moat) + TAM/SAM/SOM 시장 규모 평가
- Quick Validation 단계 (즉시 실행 가능한 검증 방법 제시)

---

## 실행 단계

### Step 0: 이전 아이데이션 결과 확인 (자동)

**Actions:**

1. `domain/side-project/market-research/` 폴더에서 이전 아이데이션 노트 검색
2. 이미 발굴/검토된 아이디어 목록 수집
3. 이 목록을 Step 2의 모든 에이전트에게 "제외 리스트"로 전달

> **왜 필요한가?** 매번 같은 아이디어가 반복 발굴되면 시간 낭비. 이전 라운드에서 이미 발견하고 버린 아이디어를 다시 제안하지 않도록 차단.

---

### Step 1: 탐색 범위 설정

사용자에게 관심 분야를 확인합니다.

**Actions:**

1. **사용자 컨텍스트 수집**

   다음을 질문하거나 기존 대화에서 추론:
   - 관심 분야 (전체 / 특정 도메인)
   - 타겟 시장 (한국 / 글로벌 / 둘 다)
   - 수익 모델 선호 (구독 / 일회성 / 광고 / 상관없음)
   - 제외할 분야 (이미 시도한 것, 관심 없는 것)
   - **NEW**: 기술 제약 (네이티브 필수? 웹 가능? Flutter OK?)

   기본값 (사용자가 별도 지정 없으면):
   ```
   - 분야: 전체 (제한 없음)
   - 시장: 한국 우선, 글로벌 고려
   - 수익 모델: 상관없음
   - 제약: 1인 개발, AI(클로드 코드) 활용
   - 기술: Android/Kotlin 우선, Flutter/웹 가능
   ```

2. **사용자 확인 후 Step 2로 진행**

---

### Step 2: 8개 에이전트 병렬 마이닝 (핵심)

**8개의 독립 에이전트**가 서로 다른 방법론으로 동시에 아이디어를 발굴합니다. Claude Code에서는 `Agent` 도구로 병렬 실행합니다.

> **왜 8개 병렬인가?** 6개 → 8개로 확장. 기존 6개 방법론에 "수익 검증"과 "실패 패턴 탐지"를 추가해 발굴-검증을 한 사이클에 완료한다.

**중요**: 모든 에이전트에게 Step 0에서 수집한 **이전 아이디어 제외 리스트**를 전달할 것.
**중요**: 각 에이전트는 **상위 5개 아이디어**를 제안 (기존 3개 → 5개로 확대).

---

**에이전트 1: Reddit/커뮤니티 마이너 (Pain Point Hunter)**

- **방법론**: Reddit Mining + 한국 커뮤니티 불만 분석
- **역할**: "사람들이 진짜 불편해하는 것"을 찾는다
- **WebSearch 검색 전략**:
  ```
  - "reddit I wish there was an app for [분야] 2025 2026"
  - "reddit frustrated with [분야] app"
  - "reddit [분야] workflow annoying"
  - "site:reddit.com [분야] need better tool"
  - "[분야] 불편 커뮤니티 site:clien.net OR site:dcinside.com OR site:fmkorea.com"
  - "site:reddit.com r/SideProject [분야]"
  - "site:reddit.com r/indiehackers what are you building"
  ```
- **수집 항목**:
  - 반복적으로 언급되는 불만/불편 (최소 7개)
  - 각 불만에 대한 추천수/공감수 (인기도 판단)
  - 현재 해결책이 없거나 부족한 문제
  - "이런 앱 없나요?" 류의 질문
  - **NEW**: 불만 발생 빈도 (일회성 vs 반복적)
- **출력**: 상위 5개 아이디어 + 근거 + 출처 링크

**에이전트 2: 앱스토어 리뷰 분석가 (Gap Finder)**

- **방법론**: App Store Review Mining + 경쟁 Gap 분석
- **역할**: 기존 인기 앱의 "빈틈"을 찾는다
- **WebSearch 검색 전략**:
  ```
  - "[분야] app reviews complaints 2025 2026"
  - "[분야] app 1 star review common"
  - "best [분야] app missing feature"

  - "Play Store [분야] app low rating reason"
  ```
- **수집 항목**:
  - 해당 분야 상위 앱 5개의 공통 불만
  - "이 기능만 있으면 완벽" 류의 리뷰
  - 별점 1-2점 리뷰에서 반복되는 패턴
  - 경쟁 앱이 무시하는 니치 사용자층
- **출력**: 상위 5개 아이디어 + 분석한 앱 목록 + Gap 근거

- **수집 항목**:
  - 급상승 키워드/토픽 (최소 5개)
  - Product Hunt에서 최근 인기 있는 도구
  - Hacker News Show HN에서 관심 받는 프로젝트
  - 아직 경쟁이 적은 신흥 카테고리
  - **NEW**: 트렌드의 성장 곡선 단계 (초기/성장/포화)
- **출력**: 상위 5개 아이디어 + 트렌드 데이터 + 타이밍 분석

- **수집 항목**:
  - 해외 1인 개발자 성공 앱 (매출 데이터 포함)

  - **NEW**: 문화적 적합성 평가 (한국에서 통할지)

  ```
  - "how much do people pay for [분야] app"
  - "앱 구독 가격 한국 시장"
  ```

- **수집 항목**:

- **출력**: 안티패턴 체크리스트 + 위험 신호 목록 (Step 4에서 감점 기준으로 활용)

### Step 3: 아이디어 통합 및 중복 제거

8개 에이전트의 결과를 통합합니다.

**Actions:**

1. **40개 아이디어 수집** (에이전트당 상위 5개 × 8개 = 40개)
   - 에이전트 7(수익 검증)과 8(안티패턴)은 아이디어 대신 **평가 기준 데이터** 제공

2. **중복/유사 아이디어 병합**
   - 여러 에이전트가 동일한 문제를 발견했으면 → **교차 검증 점수 가산**
   - 유사 아이디어는 하나로 통합하되, 각 에이전트의 관점 유지

3. **제외 필터링**
   - Step 0에서 수집한 이전 아이디어와 중복되는 것
   - 사용자가 제외한 분야
   - 1인 AI 개발로 불가능한 규모
   - 법적/규제 리스크 높은 것
   - **NEW**: 에이전트 8(안티패턴)이 식별한 함정에 해당하는 것

4. **잔여 아이디어에 수익 데이터 매핑**
   - 에이전트 7(수익 검증)의 카테고리별 수익 데이터를 각 아이디어에 연결

---

### Step 4: 점수화 및 랭킹 (강화 버전)

통합된 아이디어를 **7개 기준으로 점수화**합니다 (기존 5개 → 7개).

**평가 기준 (각 5점 만점):**

| 기준 | 설명 | 가중치 |
|------|------|--------|
| **실현 가능성** | 1인 AI 개발로 MVP 가능? 기술적 복잡도는? | ×2 |
| **수익 가능성** | 실제 수익 데이터 기반 과금 가능성? (에이전트 7 데이터) | ×2 |
| **경쟁 강도** | 경쟁이 적거나 차별화 가능? (낮을수록 높은 점수) | ×1.5 |
| **데이터 근거** | 여러 에이전트가 교차 검증? 실제 데이터? | ×1.5 |
| **경쟁 해자(Moat)** | 쉽게 복제 불가? 네트워크 효과? 전환 비용? | ×1.5 |
| **시장 규모(TAM)** | 타겟 시장 크기? 니치가 너무 작지 않은지? | ×1 |
| **안티패턴 리스크** | 에이전트 8의 실패 패턴에 해당하는지? (높을수록 감점) | ×-1 |

**가중 점수 = (실현×2 + 수익×2 + 경쟁×1.5 + 근거×1.5 + 해자×1.5 + 시장×1 - 리스크×1) / 10.5 × 100**

**출력 형식 (각 아이디어별):**

```markdown
### N위: [아이디어명] — 종합 점수: XX/100
> **한 줄:** [누가] [무슨 문제를] [어떻게 해결]

| 기준 | 점수 | 근거 |
|------|------|------|
| 실현 가능성 (×2) | X/5 | ... |
| 수익 가능성 (×2) | X/5 | 벤치마크: [앱] 월 $X |
| 경쟁 강도 (×1.5) | X/5 | ... |
| 데이터 근거 (×1.5) | X/5 | N개 에이전트 교차 발견 |
| 경쟁 해자 (×1.5) | X/5 | [해자 유형: 데이터/네트워크/브랜드/전환비용/없음] |
| 시장 규모 (×1) | X/5 | TAM: ~$Xm, SAM: ~$Xm |
| 안티패턴 리스크 (×-1) | X/5 | [해당 리스크 or "없음"] |

**발견 경로:** 에이전트 N + 에이전트 M
**해외 벤치마크:** [있으면 앱명 + 매출]
**한국 경쟁:** [있으면 앱명, 없으면 "없음"]
```
