# Ideation (앱 아이디어 발굴)

검증된 인디 해커 방법론들을 **병렬 에이전트**로 동시에 실행하여, 데이터 기반으로 앱 아이디어를 발굴하고 점수화합니다.

## 트리거 조건

다음과 같은 요청이 들어올 때 자동 발동:
- "아이데이션 해줘", "아이디어 찾아줘"
- "/ideation", "ideation"
- "뭐 만들지 찾아줘", "앱 아이디어 발굴"
- "새 사이드 프로젝트 아이디어"

## 핵심 철학

> **"좋은 아이디어는 발명하는 게 아니라 발견하는 것이다."** — Paul Graham

검증된 방법론:
- **Reddit/커뮤니티 마이닝**: 사람들이 실제로 불편해하는 것 찾기
- **앱스토어 리뷰 분석**: 기존 앱의 빈틈(Gap) 발견
- **Copy & Localize**: 해외 성공 모델을 한국화
- **Boring Business**: 섹시하진 않지만 돈 되는 틈새
- **트렌드 서핑**: 급성장 키워드에 올라타기

## 실행 단계

### Step 1: 탐색 범위 설정

사용자에게 관심 분야를 확인합니다.

**Actions:**

1. **사용자 컨텍스트 수집**

   다음을 질문하거나 기존 대화에서 추론:
   - 관심 분야 (전체 / 특정 도메인)
   - 타겟 시장 (한국 / 글로벌 / 둘 다)
   - 수익 모델 선호 (구독 / 일회성 / 광고 / 상관없음)
   - 제외할 분야 (이미 시도한 것, 관심 없는 것)

   기본값 (사용자가 별도 지정 없으면):
   ```
   - 분야: 전체 (제한 없음)
   - 시장: 한국 우선, 글로벌 고려
   - 수익 모델: 상관없음
   - 제약: 1인 개발, 퇴근 후 3-4시간, 2주 MVP
   ```

2. **사용자 확인 후 Step 2로 진행**

**모델 선택:** 메인 모델 (질문/확인만)

---

### Step 2: 6개 에이전트 병렬 마이닝 (핵심)

**6개의 독립 에이전트**가 서로 다른 방법론으로 동시에 아이디어를 발굴합니다.

> **왜 6개 병렬인가?** 하나의 방법론만 쓰면 편향된다. 커뮤니티, 앱스토어, 트렌드, 해외 벤치마크, 틈새, 기술 관점을 동시에 탐색해야 교차 검증된 기회를 발견한다.

**Actions:**

**6개 에이전트 동시 실행** (Task 도구로 병렬 실행, 각각 `sonnet`)

**에이전트 1: Reddit/커뮤니티 마이너 (Pain Point Hunter)**

- **방법론**: Reddit Mining + 커뮤니티 불만 분석
- **역할**: "사람들이 진짜 불편해하는 것"을 찾는다
- **WebSearch 검색 전략**:
  ```
  - "reddit I wish there was an app for [분야] 2025 2026"
  - "reddit frustrated with [분야] app"
  - "reddit [분야] workflow annoying"
  - "site:reddit.com [분야] need better tool"
  - "[분야] 불편 커뮤니티 site:clien.net OR site:dcinside.com OR site:fmkorea.com"
  ```
- **수집 항목**:
  - 반복적으로 언급되는 불만/불편 (최소 5개)
  - 각 불만에 대한 추천수/공감수 (인기도 판단)
  - 현재 해결책이 없거나 부족한 문제
  - "이런 앱 없나요?" 류의 질문
- **출력 형식**:
  ```markdown
  ## Reddit/커뮤니티 마이닝 결과

  ### 발견한 페인 포인트
  1. **[불만 요약]** — 출처: [링크], 공감: [수]
     - 현재 해결책: [있음/없음/부족]
     - 앱 아이디어: [제안]
  2. ...

  ### 추천 아이디어 (상위 3개)
  - 아이디어 A: ... (근거: ...)
  - 아이디어 B: ... (근거: ...)
  - 아이디어 C: ... (근거: ...)
  ```

### Step 3: 아이디어 통합 및 중복 제거 (opus)

6개 에이전트의 결과를 **opus 모델이 통합**합니다.

**Actions:**

1. **18개 아이디어 수집** (에이전트당 상위 3개 × 6개 = 18개)

2. **중복/유사 아이디어 병합**
   - 여러 에이전트가 동일한 문제를 발견했으면 → **교차 검증 점수 가산**
   - 유사 아이디어는 하나로 통합하되, 각 에이전트의 관점 유지

3. **제외 필터링**
   - 사용자가 제외한 분야
   - 1인/2주로 불가능한 규모
   - 법적/규제 리스크 높은 것

**모델 선택:** `opus` (다관점 종합 + 중복 제거 + 필터링)

---

### Step 4: 점수화 및 랭킹 (opus)

통합된 아이디어를 **5개 기준으로 점수화**합니다.

**Actions:**

**평가 기준 (각 5점 만점, 총 25점):**

| 기준 | 설명 | 가중치 |
|------|------|--------|
| **실현 가능성** | 1인/2주/퇴근 후 3-4시간으로 MVP 가능? | ×2 |
| **수익 가능성** | 사람들이 돈을 낼 의향? 수익 모델 명확? | ×2 |
| **경쟁 강도** | 경쟁이 적거나 차별화 가능? (낮을수록 높은 점수) | ×1.5 |
| **데이터 근거** | 여러 에이전트가 교차 검증? 실제 데이터? | ×1.5 |
| **성장 잠재력** | 확장 가능? v2, v3로 키울 수 있는? | ×1 |

**가중 점수 = (실현×2 + 수익×2 + 경쟁×1.5 + 근거×1.5 + 성장×1) / 8 × 100**

**출력 형식:**

```markdown
## 아이디어 랭킹 (상위 10개)

### 1위: [아이디어명] — 종합 점수: XX/100
> **한 줄:** [누가] [무슨 문제를] [어떻게 해결]

| 기준 | 점수 | 근거 |
|------|------|------|
| 실현 가능성 (×2) | ⭐⭐⭐⭐⭐ | 2주 MVP 가능, 기술 난이도 낮음 |
| 수익 가능성 (×2) | ⭐⭐⭐⭐☆ | 월 $X 과금 가능, 벤치마크: [앱] |
| 경쟁 강도 (×1.5) | ⭐⭐⭐⭐☆ | 직접 경쟁자 없음, 유사 앱 2개 |
| 데이터 근거 (×1.5) | ⭐⭐⭐⭐⭐ | 3개 에이전트 교차 발견 |
| 성장 잠재력 (×1) | ⭐⭐⭐☆☆ | 니치 시장, 확장은 제한적 |

**발견 경로:** 에이전트 1(Reddit) + 에이전트 4(해외 벤치마크)
**해외 벤치마크:** [있으면 앱명 + 매출]
**한국 경쟁:** [있으면 앱명, 없으면 "없음"]
**수익 모델 제안:** [구독/BYOK/일회성]
**기술 스택 힌트:** [간단히]

---

### 2위: [아이디어명] — 종합 점수: XX/100
...
```

**모델 선택:** `opus` (다기준 판단 + 점수 보정 + 랭킹)

### Step 5: 사용자 선택 및 다음 단계 안내

**Actions:**

1. **상위 10개 아이디어 요약 제시**

   ```markdown
   ## 발굴된 아이디어 TOP 10

   | 순위 | 아이디어 | 점수 | 핵심 매력 | 발견 경로 |
   |------|---------|------|----------|----------|
   | 1 | [이름] | XX/100 | [한 줄] | Reddit + 해외 벤치마크 |
   | 2 | [이름] | XX/100 | [한 줄] | 앱스토어 Gap |
   | ... | ... | ... | ... | ... |
   ```

2. **사용자에게 선택 요청**

   ```
   상위 10개 아이디어입니다.

   다음 중 선택해주세요:
   1. 마음에 드는 아이디어 번호 → /app-plan으로 기획 시작
   2. 특정 분야를 더 깊게 파기 → 해당 분야로 Step 2 재실행
   3. 아이디어 2개 비교 → 상세 비교 분석
   4. 전혀 새로운 방향으로 → Step 1부터 재시작
   ```

3. **선택에 따른 후속 처리**
   - 번호 선택 시: `/app-plan` 스킬 자동 연결
   - 깊게 파기: 해당 분야로 에이전트 재실행 (검색어 특화)
   - 비교: 2개 아이디어 상세 비교 테이블

## 예시

### Example 1: 분야 제한 없이 전체 탐색

**User says:**
```
아이디어 찾아줘. 분야 상관없어.
```

**Actions:**

1. **Step 1**: 기본값 사용 (전체 분야, 한국 우선, 1인/2주)

2. **Step 2**: 6개 에이전트 병렬 실행
   - 에이전트 1: "reddit I wish there was an app 2026" → "가계부 자동 분류 불편" 발견
   - 에이전트 2: "productivity app 1 star review" → "기존 투두앱 과도한 기능" 발견
   - 에이전트 3: "Product Hunt trending 2026" → "AI 요약 도구 급성장" 발견
   - 에이전트 4: "indie hacker $10K MRR" → "TypingMind BYOK 모델" 발견
   - 에이전트 5: "학원 관리 앱 불편" → "소규모 학원 출석 관리" 발견
   - 에이전트 6: "AI wrapper profitable" → "법률 문서 AI 요약" 발견

3. **Step 3**: 18개 → 중복 제거 → 14개

4. **Step 4**: 점수화 → 상위 10개 랭킹
   - 1위: AI 법률 문서 요약 (82/100) — 에이전트 3+6 교차
   - 2위: 소규모 학원 출석 관리 (78/100) — 에이전트 5 단독, 경쟁 없음
   - ...

5. **Step 5**: "마음에 드는 아이디어가 있으면 번호를 말씀해주세요!"

**Result:**
데이터 기반 10개 아이디어 + 점수 + 즉시 /app-plan 연결 가능

### Example 2: 특정 분야 지정

**User says:**
```
AI 활용 앱으로 한정해서 아이디어 찾아줘
```

**Actions:**

1. **Step 1**: 분야를 "AI 활용 앱"으로 한정
2. **Step 2**: 6개 에이전트 모두 AI 관련 검색어로 특화
   - 에이전트 1: "reddit need AI tool for [구체 분야]"
   - 에이전트 4: "AI app solo developer revenue 2026"
   - 에이전트 6: "AI API wrapper niche profitable"
3. 나머지 동일

**Result:**
AI 특화 아이디어 10개 + 점수 + BYOK/구독 수익 모델 비교

## 품질 체크리스트

### Step 2 (6개 에이전트)
- [ ] 6개 에이전트가 **독립적으로** 실행되었는가?
- [ ] 각 에이전트가 최소 3개 이상 검색했는가?
- [ ] 검색 결과에 **구체적 링크/데이터**가 포함되었는가?
- [ ] 각 에이전트가 **상위 3개 아이디어**를 제안했는가?
- [ ] 아이디어에 **근거**가 있는가? (감이 아닌 데이터)

### Step 3 (통합)
- [ ] 중복이 제거되었는가?
- [ ] 교차 발견된 아이디어에 가산점이 있는가?
- [ ] 불가능한 아이디어가 필터링되었는가?

### Step 4 (점수화)
- [ ] 5개 기준 모두 평가되었는가?
- [ ] 가중치가 올바르게 적용되었는가?
- [ ] 점수 근거가 구체적인가?
- [ ] 상위 10개가 명확하게 순위화되었는가?

### Step 5 (안내)
- [ ] 사용자에게 명확한 선택지가 제시되었는가?
- [ ] /app-plan 연결 방법이 안내되었는가?

## 트러블슈팅

### 검색 결과가 부족할 때
1. 영어 키워드로 재검색
2. 더 구체적인 분야로 좁히기
3. 시간 범위 확대 (2025~2026)
4. 대체 커뮤니티 검색 (클리앙, 디시, 블라인드)

### 모든 아이디어가 레드오션일 때
1. "Boring Business" 에이전트 결과에 집중
2. 니치를 더 좁히기 (예: "투두앱" → "소방관 교대근무 투두앱")
3. B2B 소규모 도구로 방향 전환
4. 기존 앱의 "빠진 기능 하나"만 만드는 전략

### 아이디어는 많은데 선택이 어려울 때
1. "30초 엘리베이터 테스트": 30초 안에 설명 불가 → 탈락
2. "Scratch Your Own Itch": 내가 직접 쓸 것인가?
3. "Weekend Test": 이번 주말에 바로 시작하고 싶은가?
